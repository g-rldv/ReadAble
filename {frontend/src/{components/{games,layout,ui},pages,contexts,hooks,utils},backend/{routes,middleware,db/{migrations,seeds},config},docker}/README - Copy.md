# 📚 ReadAble

**ReadAble** is a web-based interactive reading comprehension support application designed for people with special needs and reading difficulties. It uses engaging games and word activities to make reading fun, accessible, and rewarding.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🎮 **4 Game Types** | Word Match, Fill-in-the-Blank, Sentence Sort, Picture-Word |
| 📊 **Progress Tracking** | Score history, attempts, completion status |
| 🏆 **Leveling System** | Earn XP, level up every 50 XP |
| 🏅 **Achievements** | 8 unlockable badges for milestones |
| 🔊 **Text-to-Speech** | Web Speech API with adjustable voice, speed & pitch |
| 🎨 **Theme Switching** | Light, Dark, High-Contrast modes |
| 🔠 **Text Size** | Small, Medium, Large, Extra-Large |
| 📱 **Responsive** | Works on mobile, tablet, and desktop |
| 🆓 **Trial Game** | Play a mini-game without registering |
| 🔒 **JWT Auth** | Secure registration and login with bcrypt |

## 🚀 Quick Start (Local Development)

### Prerequisites
- Node.js 18+
- PostgreSQL 14+ (or a Render.com database)

### 1. Clone & Install
```bash
git clone <your-repo-url>
cd readable

# Install all dependencies
npm run install:all
```

### 2. Set Up Environment Variables
```bash
cp .env.example backend/.env
# Edit backend/.env with your DATABASE_URL and JWT_SECRET
```

### 3. Run Database Migrations & Seed Data
```bash
npm run migrate   # Creates tables
npm run seed      # Inserts activities and achievements
```

### 4. Start Development Servers
```bash
npm run dev
# Backend:  http://localhost:5000
# Frontend: http://localhost:3000
```

## 🌐 Deploy to Render.com

### One-Click Blueprint Deployment
1. Push this repo to **GitHub**
2. Go to [dashboard.render.com](https://dashboard.render.com)
3. Click **"New"** → **"Blueprint"**
4. Connect your GitHub repository
5. Render will detect `render.yaml` and create:
   - 🐘 PostgreSQL database (`readable-db`)
   - ⚙️  Backend API (`readable-backend`)
   - 🌐 Frontend static site (`readable-frontend`)
6. After deploy, open **Backend Shell** and run:
   ```bash
   node db/migrations/run.js
   node db/seeds/run.js
   ```

### Environment Variables (auto-configured by render.yaml)
| Variable | Service | Description |
|---|---|---|
| `DATABASE_URL` | Backend | Auto-injected from Render PostgreSQL |
| `JWT_SECRET` | Backend | Auto-generated by Render |
| `FRONTEND_URL` | Backend | Frontend URL for CORS |
| `VITE_API_URL` | Frontend | Backend API URL |

## 🗄 Database Schema

```sql
users          -- id, username, email, password_hash, level, xp, streak, achievements, avatar
activities     -- id, title, description, type, difficulty, content (JSONB), correct_answer (JSONB), xp_reward
user_progress  -- id, user_id, activity_id, score, attempts, completed, feedback
settings       -- id, user_id, text_size, theme, tts_enabled, tts_voice, tts_rate, tts_pitch
achievements   -- id, key, title, description, icon, condition (JSONB)
```

## 🎮 Activity Types

| Type | Description | Game Mechanic |
|---|---|---|
| `word_match` | Match words to definitions/pairs | Drag & drop |
| `fill_blank` | Choose missing words in sentences | Tap to select |
| `sentence_sort` | Order scrambled sentences | Drag & reorder |
| `picture_word` | Match emojis to words | Tap answer |

## 🔑 API Endpoints

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/api/auth/register` | ❌ | Create account |
| POST | `/api/auth/login` | ❌ | Login |
| GET | `/api/auth/me` | ✅ | Current user |
| GET | `/api/activities` | Optional | List activities |
| GET | `/api/activities/:id` | Optional | Get activity |
| POST | `/api/activities/:id/submit` | ✅ | Submit answer |
| GET | `/api/progress` | ✅ | User progress |
| GET | `/api/progress/stats` | ✅ | Aggregate stats |
| GET/PUT | `/api/settings` | ✅ | User settings |
| GET | `/api/users/leaderboard` | ❌ | Top users |
| PUT | `/api/users/avatar` | ✅ | Update avatar |
